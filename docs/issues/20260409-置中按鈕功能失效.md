# 置中按鈕功能失效問題

## 問題描述

點擊右下角浮動置中按鈕時，只有更新地址功能，但地圖不會移動到定位點位置。

## 發生時間

2026-04-09

## 根本原因

在實作「雙擊移動定位點」功能時，嘗試移除 `ChangeView` 組件以避免地圖自動置中（因為當時使用者抱怨雙擊時地圖會抖動）。但後來發現 `ChangeView` 是讓地圖跟隨定位點移動的關鍵組件。

移除 `ChangeView` 後，雖然 `requestGeolocation` 會更新 `position` 狀態，但地圖視角不會自動跟隨移動，導致置中按鈕看起來失效（實際上位置已更新，但視角未移動）。

## 修正方式

將 `ChangeView` 組件重新加回 MapContainer 內部：

```tsx
{/* 變更視角 - 當位置改變時自動置中 */}
<ChangeView center={position} zoom={zoom} />
```

`ChangeView` 組件會監聽 `position` 變化，當定位點位置改變時，自動調用 `map.setView()` 將地圖中心移動到新位置。

## 自動置中邏輯優化

### 問題分析

#### 情況 1：點擊浮動按鈕回到初始點 → 拖動定位點 → 觸發自動置中

- `requestGeolocation` 會設定 `shouldAutoCenter = true`
- 拖動定位點時，設定 `setShouldAutoCenter(true)`（已添加修正）
- 因此下一次位置變更時會觸發自動置中

#### 情況 2：右鍵改變定位點 → 拖動定位點 → 不會觸發自動置中

- 右鍵點擊時設定 `setShouldAutoCenter(false)`
- 拖動定位點時原本沒有設定 `shouldAutoCenter`，保持 `false`
- 因此後續位置變更不會觸發自動置中

### 差異原因

| 情境 | shouldAutoCenter 狀態 |
|------|----------------------|
| 浮動按鈕定位 | 設為 `true` → 拖動後保持 `true` → 會置中 |
| 右鍵移動定位 | 設為 `false` → 拖動後保持 `false` → 不會置中 |

### 修正方式

在拖動定位點的 `dragend` 事件中，設定 `setShouldAutoCenter(false)`：

```tsx
eventHandlers={{
    dragend: (e) => {
        const latlng = e.target.getLatLng();
        /** 拖曳定位點後，不自動置中（打斷瀏覽體驗） */
        setShouldAutoCenter(false);
        setPosition([latlng.lat, latlng.lng]);
        setLocationError(false);
        updateAddress(latlng.lat, latlng.lng);
    },
}}
```

### 設計邏輯

- **右鍵移動定位點**：使用者明確選擇了一個位置，不應打斷其瀏覽體驗 → 不自動置中 (`false`)
- **拖動定位點**：使用者正在調整位置，不應打斷瀏覽體驗 → 不自動置中 (`false`)
- **浮動按鈕/GPS 定位**：明確要求回到自己所在位置 → 自動置中 (`true`)

## 預防建議

未來在移除或修改任何功能前，應先確認該功能的所有依賴關係，避免破壞其他功能。

## 最終行為確認 (2026-04-09)

經過修正後，所有功能正常運作：

| 操作方式 | shouldAutoCenter | 是否觸發自動置中 |
|----------|-----------------|-----------------|
| 浮動按鈕/GPS 定位 | `true` | ✅ 是 |
| 右鍵移動定位點 | `false` | ❌ 否 |
| 拖動定位點 | `false` | ❌ 否 |
| 手動模式點擊地圖 | `false` | ❌ 否 |
