# AGENTS.md

此檔案定義本專案的 agents 行為規範。

## 重要原則

1. **當已經決定要編輯或建立檔案時，請用 EDIT 編輯工具，而不是將代碼印出來在對話內**

2. **禁止主動操作 Git 相關行為**
   - 除非明確指示，否則不要執行 `git commit`、`git push`、`git pull`、`git merge` 等操作
   - 不要主動建議提交，除非使用者明確要求
   - 不要主動建立分支、切換分支

3. **除非有需要比對歷史來比較變化，否則非必要請不要一直操作 GIT**

---

## 檔案編輯原則

- 若需要建立新檔案，使用 `Write` 工具
- 若需要修改現有檔案，使用 `Edit` 工具
- 避免在對話中直接顯示完整程式碼，應直接操作檔案

## 執行原則

- **主動實際執行**：當需要編輯或建立檔案時，直接使用工具操作，不要只停留在對話中說明
- **不依賴使用者動手**：禁止要求使用者自行編輯或建立檔案，所有操作應由 agents 實際執行
- **建立 TODO 清單**：執行任何任務時，應使用 `todowrite` 工具建立 TODO 清單
- **追蹤進度**：將任務分解為具體的 TODO 項目，標記進度狀態
- **保持簡潔**：每個階段保持 3-5 個項目，複雜任務分階段執行
- **除非問答或規劃模式否則實際編輯**：除非處於問答模式或規劃模式，否則請實際編輯檔案
- **確認任務完成**：任務結束時，請確認檔案確實編輯完成
- **複查成果**：每完成一個階段或任務結束時，請呼叫 shadow-sovereign 複查成果

## Implementation Mode（實作模式）

當處於實作模式時，嚴格遵守以下規則：

- **無對話贅字**：不要使用「Sure, I can help」、「Here is the code」等制式开场白
- **直接產生代碼**：直接提供檔案路徑和完整實作內容
- **標準 JSDoc**：所有函式使用標準 JSDoc 註解，確保 AI 可讀性
- **優先使用工具**：若環境支援工具呼叫，優先使用 Write 工具輸出而非文字回覆
- **說明化為註解**：如需說明，將內容以註解形式置於程式碼內，而非對話中
- **預設模式**：請記住你的身份預設為 Implementation Mode（實作模式）

## 測試與除錯

- **使用 Chrome DevTools**：可以使用 `chrome-devtools` 工具察看待測網站來除錯或測試操作網站內容
- **處理無法連線**：如果測試網址無法連線，代表可能沒有執行 `dev` 指令或 `dev` 啟動失敗
- **開發伺服器 Hot Reload**：開發伺服器基本上有 hot reload 功能，不太需要主動重啟
- **謹慎請求重啟**：如果確定更改無誤，卻又看不見變化或修正問題，可以請示使用者重啟伺服器
- **主動排查問題**：代理應自行查看測試伺服器、自行檢查 console 訊息、自行排查，無法做到時才請求使用者協助
- **主動完成任務**：當知道要做什麼的時候，應該要自己主動去完成，而不是要求使用者去處理（例如：請使用者重新整理頁面、請使用者檢查 console）
- **禁止空談不做事**：禁止只是給出大量說明文字或程式碼，卻沒有實際去驗證問題是否解決、沒有實際去排查除錯
