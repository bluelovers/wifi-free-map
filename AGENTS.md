# AGENTS.md

此檔案定義本專案的 agents 行為規範。

## 重要原則

1. **當已經決定要編輯或建立檔案時，請用 EDIT 編輯工具，而不是將代碼印出來在對話內**

2. **禁止主動操作 Git 相關行為**
   - 除非明確指示，否則不要執行 `git commit`、`git push`、`git pull`、`git merge` 等操作
   - 不要主動建議提交，除非使用者明確要求
   - 不要主動建立分支、切換分支

3. **除非有需要比對歷史來比較變化，否則非必要請不要一直操作 GIT**

---

## 檔案編輯原則

- 若需要建立新檔案，使用 `Write` 工具
- 若需要修改現有檔案，使用 `Edit` 工具
- 避免在對話中直接顯示完整程式碼，應直接操作檔案

## 執行原則

- **主動實際執行**：當需要編輯或建立檔案時，直接使用工具操作，不要只停留在對話中說明
- **不依賴使用者動手**：禁止要求使用者自行編輯或建立檔案，所有操作應由 agents 實際執行
- **建立 TODO 清單**：執行任何任務時，應使用 `todowrite` 工具建立 TODO 清單
- **追蹤進度**：將任務分解為具體的 TODO 項目，標記進度狀態
- **保持簡潔**：每個階段保持 3-5 個項目，複雜任務分階段執行
